Fundamento

O que é SDLC?

A definição abaixo é a adotada pela própria documentação do GitHub.

SDLC significa Software Development Life Cycle — ciclo de vida de desenvolvimento de software.

É um “processo estruturado, passo a passo, que ajuda equipes de desenvolvimento a criar software de alta qualidade com eficiência e menor custo. As equipes usam o SDLC para planejar, analisar, projetar, testar, implantar e manter o software sistematicamente”, garantindo que ele atenda aos requisitos das partes interessadas e aos padrões de qualidade, segurança e conformidade da organização. [1]

💡
Existem várias metodologias para organizar o SDLC — Cascata (Waterfall), Ágil, Iterativo, Espiral, modelo V e Lean. O DevOps é descrito pelo GitHub como uma abordagem do SDLC que combina Dev e Ops, apoiada em automação, segurança e CI/CD. [1]
As 6 fases

As fases do SDLC e como o Copilot atua em cada uma

As fases e suas atividades seguem o artigo oficial do GitHub [1]. Os recursos do Copilot indicados em cada fase vêm da documentação oficial do produto [2][3][4].

1

Coleta e análise de requisitos

Levantar, avaliar e documentar requisitos precisos, completos e mensuráveis.

📋 Atividades da fase [1]
  • Levantar requisitos via entrevistas, workshops e questionários; observar como as partes interessadas trabalham.
  • Avaliar a viabilidade do sistema e registrar em histórias de usuário, SRS ou documentos de caso de uso.
✦ Onde o GitHub Copilot ajuda

Conversar em linguagem natural para explorar requisitos e organizar contexto do problema antes de codificar.

Copilot Chat [3] Copilot Spaces [3] Custom instructions [3]
2

Projeto do sistema (design)

Traduzir requisitos em arquitetura, modelos e um desenho modular.

📋 Atividades da fase [1]
  • Modelar comportamento e estrutura com UML, fluxogramas, árvores/tabelas de decisão e simulações.
  • Aplicar separação de preocupações para um projeto modular, mais fácil de testar e manter.
✦ Onde o GitHub Copilot ajuda

Gerar grandes seções a partir de uma ideia e iterar sobre elas; atribuir uma persona (ex.: “desenvolvedor sênior focado em qualidade”) para discutir trade-offs de design.

Copilot Chat [4] Custom instructions [3] Prompt files [3]
3

Codificação

Transformar as especificações de projeto em código limpo, legível e versionado.

📋 Atividades da fase [1]
  • Escrever código fácil de ler, comentar o que ele faz e usar controle de versão para rastrear mudanças.
  • Refatorar quando necessário e realizar code review ao concluir.
✦ Onde o GitHub Copilot ajuda

É o coração do produto: sugestões de autocompletar enquanto você digita, geração de código a partir de comentários, e execução autônoma de tarefas dentro da IDE.

Inline suggestions [2] Next edit suggestions [2] Agent mode na IDE [2] Copilot CLI [2] Explicar/comentar código [4]
4

Testes

Validar exaustivamente o software antes de ir para produção.

📋 Atividades da fase [1]
  • Definir plano e casos de teste; executar testes unitários, de integração, de sistema e de aceitação.
  • Usar ferramentas automatizadas para feedback rápido e comparação com os resultados esperados.
✦ Onde o GitHub Copilot ajuda

Escrever testes está entre as tarefas que o Copilot “faz melhor”; as sugestões inline são indicadas para gerar testes em test-driven development.

Geração de testes [4] Inline suggestions (TDD) [4] Debug e correção de sintaxe [4]
5

Implantação (deploy)

Levar o código ao ambiente de produção com automação e gestão de versão.

📋 Atividades da fase [1]
  • Confirmar o código no repositório; a automação de implantação dispara uma série de testes.
  • Disponibilizar o software aos usuários com release management e controle de versão estáveis.
✦ Onde o GitHub Copilot ajuda

Agentes podem adicionar funcionalidades ou corrigir bugs e abrir um pull request ao final — conectando o trabalho à automação de entrega (o GitHub cita o GitHub Actions como ferramenta de automação do SDLC).

Copilot CLI → cria PR [2] Copilot cloud agent [2] GitHub Actions (SDLC) [1]
6

Manutenção e suporte

Monitorar, corrigir bugs, tratar segurança e evoluir o software já em produção.

📋 Atividades da fase [1]
  • Emitir correções periódicas para bugs e problemas de segurança; monitorar o desempenho técnico e percebido.
  • Fornecer treinamento e documentação aos usuários.
✦ Onde o GitHub Copilot ajuda

Revisão de código assistida por IA, resumos de pull request para o revisor focar no que importa, e diagnóstico/correção de defeitos.

Copilot code review [2] PR summaries [2] Debugging [4] Copilot Memory [3]

Catálogo oficial

As 4 categorias de recursos do GitHub Copilot

A documentação oficial organiza os recursos do Copilot nestas famílias. [2]

Recursos assistivos ASSISTIVE

Usados de forma síncrona, oferecendo conselhos ou sugestões enquanto a pessoa trabalha.

💬 Copilot Chat

Interface de chat para tirar dúvidas sobre código na IDE, no GitHub, no Mobile e no Windows Terminal.

⌨️ Inline suggestions

Sugestões de autocompletar; em VS Code, Xcode e Eclipse há ainda next edit suggestions.

📝 PR summaries

Resumos gerados por IA das mudanças de um pull request e do que o revisor deve observar.

🖥 Copilot no GitHub Desktop

Geração automática de mensagens e descrições de commit a partir das alterações.

Recursos agênticos AGENTIC

Trabalham de forma autônoma, em geral pedindo aprovação humana para ações sensíveis (rodar comandos, fazer merge).

🤖 Agent mode na IDE

O Copilot decide quais arquivos alterar, propõe mudanças e comandos para aprovação e itera até concluir.

☁️ Copilot cloud agent

Agente autônomo que pesquisa o repositório, cria um plano, altera código em um branch e abre PR.

⌨️ Copilot CLI

Adiciona features ou corrige bugs no terminal e cria pull requests.

🔍 Copilot code review

Sugestões de revisão geradas por IA para escrever um código melhor.

Recursos de personalização CUSTOMIZATION

Adicionam contexto e melhoram o desempenho do Copilot conforme as necessidades da equipe.

📦 Copilot Spaces

Centralizam código, docs e specs para ancorar as respostas no contexto certo.

📐 Custom instructions

Detalham preferências, ferramentas e requisitos para respostas mais alinhadas.

📄 Prompt files & Agent skills

Instruções reutilizáveis e pacotes de scripts/recursos para tarefas especializadas.

🔌 MCP servers

Conectam o Copilot a ferramentas e fontes de dados externas via Model Context Protocol.

Recursos para administradores ADMIN

Disponíveis a donos de organização/enterprise com planos Copilot Business ou Enterprise.

🛡 Policy & Access management

Gestão de políticas e de quem pode usar o Copilot na organização ou enterprise.

📊 Usage data

Dados de uso para orientar gestão de acesso e impulsionar a adoção.

🧾 Audit logs

Registros de auditoria para entender quais ações foram tomadas e por quem.

🚫 File exclusions

Configuram arquivos que o Copilot deve ignorar.

Governança

Boas práticas oficiais ao aplicar o Copilot no SDLC

Diretrizes da página oficial de boas práticas do GitHub. [4]

🎯 Para extrair o máximo — com responsabilidade

  • Entenda forças e limites. O Copilot brilha em escrever testes e código repetitivo, depurar e corrigir sintaxe, explicar/comentar código e gerar expressões regulares. Ele não substitui sua expertise — “você está no comando”.
  • Escolha a ferramenta certa. Sugestões inline para completar trechos e gerar testes (TDD); Copilot Chat para perguntas em linguagem natural e geração de grandes blocos com iteração.
  • Crie prompts cuidadosos. Quebre tarefas complexas, seja específico nos requisitos, dê exemplos de entrada/saída e siga boas práticas de código.
  • Verifique o trabalho do Copilot. Entenda a sugestão antes de usar, revise funcionalidade, segurança, legibilidade e manutenibilidade, e use testes automatizados, linting e code scanning como camada extra de verificação.
  • Conduza para boas saídas. Forneça contexto útil (abra arquivos relevantes, feche os irrelevantes), reescreva prompts que não ajudaram e dê feedback (aceitar/rejeitar, 👍/👎).
🔐
O artigo de SDLC do GitHub reforça que a plataforma verifica o código em busca de problemas de segurança à medida que ele é escrito, na fase de codificação — alinhando o uso de IA a um SDLC seguro. [1]
