Lab 1 — Analyze an image with Azure AI Vision (Guided)

Objective

Call Azure AI Vision from Python to caption, tag, and detect objects in an image. By the end you'll understand the request/response shape and which features cost extra calls.

Why it matters for AI-900

Domain 3 expects you to distinguish Computer Vision from Custom Vision and from Document Intelligence. Doing the call once removes the abstraction and makes the trade-offs obvious.

Prerequisites

  • Azure subscription (free tier works — free account)
  • Python 3.9+ installed (python --version)
  • Azure CLI (az --version) — install
  • Review prereqs in fundamentals.html if "endpoint", "key", or "resource group" feel unfamiliar

Step 1 — Create the resource

Why: Every Azure AI service is provisioned as a resource that gives you an endpoint URL and an authentication key.

# sign in
az login

# create a resource group (region near you)
az group create --name rg-ai900-vision --location eastus

# create an Azure AI Vision (Computer Vision) resource
az cognitiveservices account create \
  --name ai900-vision-demo \
  --resource-group rg-ai900-vision \
  --kind ComputerVision \
  --sku F0 \
  --location eastus \
  --yes
⚠️ F0 is the free tier: 5,000 transactions / month, 20 calls / min. Plenty for this lab. Limit one free resource per subscription per region.

Step 2 — Get endpoint and key

Why: Your Python script needs both to authenticate REST calls.

az cognitiveservices account show \
  --name ai900-vision-demo --resource-group rg-ai900-vision \
  --query "properties.endpoint" --output tsv

az cognitiveservices account keys list \
  --name ai900-vision-demo --resource-group rg-ai900-vision \
  --query "key1" --output tsv

Copy both values into a .env file (see .env.example).

Step 3 — Set up Python environment

cd labs/01-azure-ai-vision
python -m venv .venv
.\.venv\Scripts\Activate.ps1   # PowerShell on Windows
# or: source .venv/bin/activate   # bash
pip install -r requirements.txt

Step 4 — Run the analysis

Why: A single REST call can return tags, caption, objects, dense captions, people, smart crops, and OCR (Read). Choose features carefully — each is billed separately.

python analyze.py https://learn.microsoft.com/azure/ai-services/computer-vision/media/quickstarts/presentation.png

Expected output: JSON-formatted caption + list of tags + bounding boxes per detected object.

Step 5 — Read the code

Open analyze.py. Key elements:

  • ImageAnalysisClient — official SDK (docs)
  • visual_features=[CAPTION, TAGS, OBJECTS] — every feature is a separate billable analysis
  • AzureKeyCredential — auth header; could be swapped for DefaultAzureCredential for managed-identity scenarios (Responsible AI principle: Privacy & Security)

Success criteria

  • Script runs without exceptions
  • Caption text matches the image content meaningfully
  • You can name 3 visual features you DID NOT request (e.g., READ for OCR, PEOPLE, DENSE_CAPTIONS, SMART_CROPS)
  • You understand why you'd pick this service over Custom Vision (no training needed) and over Document Intelligence (not optimized for forms)

Troubleshooting

ErrorLikely causeFix
401 PermissionDeniedWrong key or endpoint mismatchRe-copy key1; check endpoint trailing slash
InvalidImageUrlImage needs to be publicly accessible HTTPSUse the sample URL or upload to a public blob
UnsupportedRegionFeature not in that regionUse eastus, westus2, westeurope
429 TooManyRequestsF0 rate limit hitWait 60s or upgrade to S1
ModuleNotFoundErrorWrong venv activepip install -r requirements.txt in the right venv

What this teaches about the exam

  • You can now distinguish capabilities (caption / tags / objects / OCR) — high-frequency exam content for D3
  • The F0 free tier exists for fundamentals — useful trivia
  • Pre-built models require no training data, unlike Custom Vision
  • The same SDK pattern (resource → endpoint → key → client → call) repeats across every Azure AI service

Clean up

az group delete --name rg-ai900-vision --yes --no-wait
