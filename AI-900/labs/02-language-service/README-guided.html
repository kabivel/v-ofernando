Lab 2 — Sentiment + entity extraction (Guided)

Objective

Call Azure AI Language to analyze a batch of product reviews: sentiment, key phrases, named entities, language detection. By the end you'll know which feature returns what and how they're billed per document.

Why it matters for AI-900

Domain 4 expects you to match NLP features to use cases. The single biggest exam trap is confusing AI Language (text analytics) with AI Speech (audio) and with AI Translator (text translation). Running real calls fixes the confusion.

Prerequisites

  • Azure subscription
  • Python 3.9+ & Azure CLI
  • Familiarity with NER, sentiment definitions

Step 1 — Create the Language resource

az login
az group create --name rg-ai900-lang --location eastus

az cognitiveservices account create \
  --name ai900-lang-demo \
  --resource-group rg-ai900-lang \
  --kind TextAnalytics \
  --sku F0 \
  --location eastus \
  --yes
⚠️ --kind TextAnalytics is correct for Azure AI Language despite the legacy name. The F0 free tier handles 5,000 transactions / month.

Step 2 — Endpoint + key

az cognitiveservices account show \
  -n ai900-lang-demo -g rg-ai900-lang \
  --query properties.endpoint -o tsv

az cognitiveservices account keys list \
  -n ai900-lang-demo -g rg-ai900-lang \
  --query key1 -o tsv

Step 3 — Python environment

cd labs/02-language-service
python -m venv .venv && .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Step 4 — Run the analysis

python analyze.py

Expected output (per review): detected language · sentiment (positive/neutral/negative + confidences) · key phrases · named entities with categories.

Step 5 — Read the code

Open analyze.py. Key elements:

  • TextAnalyticsClient SDK — same auth pattern as Lab 1 (docs)
  • Each method (analyze_sentiment, extract_key_phrases, recognize_entities, detect_language) is a separate billable transaction per document
  • The service auto-detects language but you can pass language="en" for a small latency win

Success criteria

  • All 4 reviews scored without exceptions
  • You can identify at least one entity category returned (e.g., Organization, Person, DateTime, Quantity)
  • You can name 2 features of AI Language you did NOT call (e.g., PII detection, summarization, CLU, Question answering)

What this teaches about the exam

  • One resource, many features — AI Language is unified (sentiment + NER + key phrases + PII + summarization + CLU + Q&A all from one endpoint)
  • Same pattern as AI Vision in Lab 1 → reinforces "endpoint + key + client + call"
  • Reading entity categories (e.g., PersonType, Quantity, DateTime) gives you concrete answers for NER questions

Clean up

az group delete --name rg-ai900-lang --yes --no-wait
