Lab 3 — Chat completion with Azure OpenAI (Guided)

Objective

Deploy a chat model in Azure AI Foundry, then call it from Python. You'll see system message, user message, temperature, max_tokens, and the content filter result fields with your own eyes.

Why it matters for AI-900

Domain 5 (Generative AI) is the heaviest at 20–25%. You cannot reason confidently about prompt engineering, tokens, or content filters without running at least one chat call.

Prerequisites

  • Azure subscription with access to Azure OpenAI — some subscription types require application approval
  • Python 3.9+ & Azure CLI
  • ~10 min for resource provisioning
⚠️ If your subscription does not have Azure OpenAI access, use the Foundry playground only (no code needed). You can skip steps 3–5 and complete this lab using the UI playground; the rest of the exam knowledge transfers.

Step 1 — Provision Azure OpenAI

Recommended: use Foundry portal — it provisions Azure OpenAI + a hub + a project in one flow.

  1. Open ai.azure.com → + New project
  2. Pick a region with the model you want (e.g., eastus for gpt-4o-mini)
  3. In your project → Deployments → + Deploy model → gpt-4o-mini → default settings → name it chat

Or via CLI:

az group create -n rg-ai900-oai -l eastus
az cognitiveservices account create \
  -n ai900-oai-demo -g rg-ai900-oai \
  --kind OpenAI --sku S0 -l eastus --yes
az cognitiveservices account deployment create \
  -n ai900-oai-demo -g rg-ai900-oai \
  --deployment-name chat \
  --model-name gpt-4o-mini --model-version "2024-07-18" \
  --model-format OpenAI --sku-name Standard --sku-capacity 1

Step 2 — Get endpoint + key

az cognitiveservices account show -n ai900-oai-demo -g rg-ai900-oai \
  --query properties.endpoint -o tsv
az cognitiveservices account keys list -n ai900-oai-demo -g rg-ai900-oai \
  --query key1 -o tsv

Step 3 — Python environment

cd labs/03-azure-openai
python -m venv .venv && .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

Step 4 — Run the chat

python chat.py "What are the 6 Microsoft Responsible AI principles?"

Expected: model lists Fairness, Reliability & Safety, Privacy & Security, Inclusiveness, Transparency, Accountability — with brief definitions. Token usage printed at the end (prompt_tokens + completion_tokens + total_tokens).

Step 5 — Experiment

  1. Change temperature from 0 → 1.0. Re-run. Compare creativity / consistency.
  2. Lower max_tokens to 30. Observe truncation.
  3. Replace the system message with "You are a sarcastic teenager". Observe persona shift.
  4. Send a benign question with personal data: "My phone 555-0100, redact PII". Inspect content-filter results.

Step 6 — Read the code

Open chat.py. Key elements:

  • AzureOpenAI client takes azure_endpoint + api_key + api_version
  • model="chat" = your deployment name, not the model name (common trap!)
  • Messages: system (persona) → user (question) → assistant (history if multi-turn)
  • usage contains token counts — directly useful for cost estimation

Success criteria

  • Script returns a completion grounded in known facts
  • You can explain why model param = deployment name, not model name
  • You observed temperature affecting output variability
  • You can name the 4 content-filter categories (hate, sexual, violence, self-harm)

Troubleshooting

ErrorCauseFix
404 DeploymentNotFoundWrong model paramPass deployment name (e.g., chat) — not gpt-4o-mini
401Wrong key or endpointRe-copy both
403 Access deniedSubscription not approved for Azure OpenAIApply at aka.ms/oai/access or use Foundry playground only
content_filter_result triggeredPrompt or completion hit a category thresholdRephrase, or adjust filter (requires approval for some categories)
429Tokens-per-minute (TPM) capacity exhaustedIncrease deployment capacity or wait

What this teaches about the exam

  • Deployment name ≠ model name — a top exam confusion
  • System message shapes behavior; user message asks the question
  • Tokens are billed for prompt + completion — context-window includes both
  • The content filter is on by default for 4 harm categories
  • Azure OpenAI = OpenAI models + Microsoft enterprise controls (RBAC, regional, SLA, private networking)
