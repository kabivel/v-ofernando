Fundamentals — pre-requisite primer

AI-900 has no formal prerequisites, but candidates do better with awareness of cloud basics and a few math/ML concepts. If your diagnostic pre-requisite score is < 30%, spend Day 1 on this primer before touching domains.

2. AI vocabulary (must know before Day 2)

TermWhat it means
AI vs MLAI = systems that mimic intelligence; ML = subset of AI that learns from data without explicit rules
Supervised learningTrain on labeled examples (input → known output). Used for regression, classification.
Unsupervised learningFind structure in unlabeled data. Used for clustering, anomaly detection.
Features / labelsInputs (features) and the target value you want to predict (label)
Training / validation / test splitTypically 70 / 15 / 15 — never test on data the model saw during training
InferenceUsing a trained model to predict on new data (also called "scoring")
Foundation model / LLMLarge pre-trained model adapted to many downstream tasks (e.g., GPT-4o)
Prompt / completion / tokenInput text → generated text; billed per token (~4 chars EN)

Reference: Get started with AI on Azure (learn path)

3. Statistics primer (light)

Term1-line definition
Accuracy(TP+TN) / total — how often the model is right
PrecisionTP / (TP+FP) — of items predicted positive, how many actually are
RecallTP / (TP+FN) — of actual positives, how many we caught
F1 scoreHarmonic mean of precision and recall
Confusion matrixTable of TP / FP / FN / TN counts for classification
RMSE / MAERegression error metrics — lower is better
OverfittingModel memorizes training data, performs poorly on new data
