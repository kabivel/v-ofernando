2. Evaluation metrics

MetricMeaningRange

4. Typical ML workflow

  1. Data prep — collect, clean, label, split into train / validation / test (~70/15/15)
  2. Train — fit an algorithm to training data
  3. Evaluate — compute metrics on validation set; iterate
  4. Test — final unbiased estimate on test set (use once!)
  5. Deploy — register model + create endpoint
  6. Monitor — watch for data drift / accuracy decay; retrain if needed

5. Self-check

  1. You want to predict tomorrow's temperature in °C. Which task type?
  2. You group 10,000 customers by purchasing behavior without labels. Which task type?
  3. Your fraud-detection classifier shows 99% accuracy but catches only 10% of frauds. Which metric should you prioritize instead?
  4. You want to train an ML model without writing code. Which Azure ML feature?
  5. What does RMSE measure?

Answers: 1. Regression · 2. Clustering (unsupervised) · 3. Recall (or F1) · 4. Designer or AutoML · 5. Root mean square error of regression predictions (lower is better)
